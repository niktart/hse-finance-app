Паттерны GoF 
Порождающие:
Фабрика - FinancialFactory - создание объектов с валидацией

DI-контейнер - DIContainer - управление зависимостями

Структурные:

Декоратор - TimingCommandDecorator - измерение времени

Прокси - CachedFinancialService - кэширование данных

Компоновщик - ICommand - единый интерфейс команд


Поведенческие:
Команда - команды операций (CreateAccountCommand)

Шаблонный метод - DataImporter - алгоритм импорта

Принципы SOLID/GRASP
SOLID:
S: Каждый класс имеет одну ответственность

O: Легко расширяется новыми командами, сериализаторами

L: CachedFinancialService заменяет FinancialService

I: Интерфейсы минимальны и специфичны

D: Зависимости через интерфейсы + DI-контейнер

GRASP:
High Cohesion: Логика разделена по ответственности

Low Coupling: Связи через интерфейсы, DI-контейнер

DI-контейнер
Реализован DIContainer с регистрацией и разрешением зависимостей
Используется для FinancialService, CachedFinancialService, FinancialFactory
